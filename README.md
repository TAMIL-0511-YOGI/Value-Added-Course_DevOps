# 🖥 Virtualization, Linux & Docker Practice Guide

This repository contains step-by-step exercises and tasks to help students set up their development environment, practice Linux basics, and explore Docker.

- **Task-1.md** → Dev Environment Setup (VS Code, Git Bash)
- **Task-2-VB-Install.md** → VirtualBox Installation Guide
- **Task-3-Vagrant.md** → Vagrant Setup with VirtualBox
- **Practice-1-Linux.md** → Linux Hands-On Exercises
- **Practice-2-Docker.md** → Docker Basics & Challenges

---

## 🚀 Learning Path

### 1️⃣ Environment Setup
- Install **VS Code** for editing scripts.
- Install **Git Bash** for terminal commands.
- Install **VirtualBox** (with VC++ Redistributable if required).
- Install **Vagrant** and verify installation.

### 2️⃣ Virtualization & Vagrant
- Fork and clone the provided GitHub repo.
- Start VM with `vagrant up --no-provision`.
- Connect via `vagrant ssh`.
- Verify OS version with `lsb_release -a`.

### 3️⃣ Linux Practice
- Create directories and files (`mkdir`, `touch`).
- Move, copy, and delete files (`mv`, `cp`, `rm`).
- Work with permissions (`chmod`).
- Search text (`grep`, `find`).
- Write simple scripts (automation, networking checks).
- Complete beginner → advanced challenges.

### 4️⃣ Docker Practice
- Check Docker installation (`docker --version`).
- Work with images (`docker pull`, `docker rmi`).
- Run containers (`docker run`, `docker ps`).
- Explore lifecycle commands (`stop`, `start`, `rm`).
- Practice bind mounts, port mapping, and Dockerfiles.
- Complete challenge tasks (compose stack, build custom images, troubleshoot).

---

## 🎯 Suggested Workflow

1. **Start with Task-1, Task-2, Task-3** → Environment ready.
2. **Do Practice-1-Linux** → Build confidence with shell commands.
3. **Move to Practice-2-Docker** → Learn containerization basics.
4. **Attempt Challenges** → Apply knowledge in real-world scenarios.

---

## 🏁 Tips for Students
- Run commands step by step and verify outputs.
- Use `man <command>` or `--help` for documentation.
- Keep notes of errors and fixes — they’re part of the learning process.
- Try challenges without hints first, then check hints if stuck.

---

## 📌 Instructor Notes
- Encourage students to explain each command before running it.
- Use checkpoints to ensure progress.
- Promote experimentation with variations of commands.
